# Harmonogram projektu programu „Pogoda”

| Etap | Zadanie | Czas |
|------|---------|------|
| Analiza wymagań | Określenie funkcjonalności programu: pobieranie temperatury i prędkości wiatru dla wybranego miasta, możliwość wyboru jednostek oraz obsługa trybu CLI i GUI | 2h |
| Projektowanie | Zaprojektowanie struktury programu oraz podział na funkcje (`get_weather`, `show_gui`, `main`) | 1h |
| Projektowanie | Zaplanowanie sposobu komunikacji z serwisem wttr.in oraz formatu przetwarzania danych JSON | 2.5h |
| Implementacja | Przygotowanie struktury programu oraz import wymaganych bibliotek | 0.5h |
| Implementacja | Implementacja funkcji `get_weather()` pobierającej dane pogodowe z API | 3h |
| Implementacja | Dodanie przetwarzania danych JSON i konwersji jednostek temperatury oraz prędkości wiatru | 2h |
| Implementacja | Implementacja obsługi argumentów wiersza poleceń przy użyciu biblioteki `argparse` | 1.5h |
| Implementacja | Stworzenie graficznego interfejsu użytkownika w `tkinter` | 3-5h |
| Wdrożenie | Przygotowanie programu do uruchomienia oraz instalacja wymaganych bibliotek | 1h |
| Dokumentacja | Przygotowanie dokumentacji programu i instrukcji użytkowania | 1.5h |
